# legacy-md-indexer

## Overview
High-throughput metadata extraction and cataloging pipeline for 9,000+ unstructured legacy documents. Migrated from a 32B VLM architecture to a lightweight text-LLM setup utilizing pre-parsed Markdown (`processed_md/`) to resolve inference bottlenecks and maximize tk/s.

## Phase 0: Document ETL Pipeline (Pre-requisite)
Transforms raw hybrid legacy documents (PDF, DOCX, PPTX, XLSX, ZIP) into structured Markdown, strictly preserving table layouts and document hierarchy for optimal LLM ingestion.
- **Automated Ingestion:** Recursive directory scanning with automatic `.zip` archive extraction.
- **Vision & OCR Integration:** Built-in multi-language (KO/EN) OCR via `EasyOCR` and `docling` for scanned documents.
- **Strict Memory Management:** Prevents RAM/VRAM leaks during massive batch conversions using `multiprocessing.Pool` with isolated worker limits (`maxtasksperchild=50`) and lazy engine initialization.
- **Idempotent Execution:** State-aware processing safely skips existing `.md` files, ensuring seamless resume upon interruption.

## Architecture
- **2-Track Routing System**
  - **Track 1 (Regex Fast-track):** Directory and filename pattern matching. Handles whitespace normalization and date pattern extraction (e.g., YY_MM_DD) with zero LLM compute overhead.
  - **Track 2 (LLM Fallback):** Handles Track 1 misses. Utilizes 8B-14B LLMs for zero-shot dynamic taxonomy generation (e.g., extracting novel categories like blueprints or estimates from raw text).
- **Output:** `file_catalog.json` containing `doc_type`, `year`, and `month`.

## System Engineering & Stability
- **Crash-safe State Management:** Synchronous JSON dump executed immediately after single-file processing. Guarantees zero data loss and instant resume capability upon SIGKILL, system crash, or OOM.
- **Resource Isolation & OOM Defense:** Explicit `CUDA_VISIBLE_DEVICES` allocation. Enforces strict context window constraints (1,500-2,000 chars per file) and blocks File Descriptor (FD) leaks.
- **CoT Parser Defense:** Pre-emptive regex stripping of `<think>...</think>` tags generated by reasoning models to prevent JSON decode failures.

## Future Scope
- **MCP & Agentic Router:** Function calling integration to parse natural language queries into specific search parameters.
- **Pre-filtered RAG:** Multi-stage retrieval. Queries `file_catalog.json` first to isolate target documents before chunk injection, strictly preventing LLM hallucination.
